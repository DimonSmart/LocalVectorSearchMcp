# 0002.spec-configuration-and-security

## Intent

The server is configured through either YAML or server CLI arguments and must fail with understandable errors when configuration or path access is invalid.

## Related Specifications

- 0001.spec-product-overview.md
- 0004.spec-index-storage-and-search.md
- 0005.spec-mcp-and-cli.md

## Behavior

The server has built-in defaults and does not require a YAML file. A configuration path may be provided with `--config`; when it is not provided, `local-vector-search-mcp.yml` is not loaded implicitly. If `--config` is provided, the file must exist. Effective configuration is built by applying default configuration, then explicit YAML configuration when present, then CLI overrides.

Configuration includes server, storage, embedding, chunking, search, and a singular `knowledgeBase` section. `knowledgeBase.root` may be omitted to use the detected project root. Names or lists of knowledge bases are not supported. A legacy top-level `knowledgeBases` section is rejected with a clear migration error. The default embedding endpoint is `http://localhost:11434/v1`, provider is `openai-compatible`, API key defaults to `ollama`, model defaults to `bge-m3:latest`, batch size defaults to 16, and remote endpoints are rejected unless explicitly allowed.

The detected project root is `CLAUDE_PROJECT_DIR` when it is non-empty, otherwise the current working directory. The knowledge base root is CLI `--root`, otherwise `knowledgeBase.root` from YAML, otherwise the project root. The storage path is CLI `--storage-path`, otherwise `storage.path` from YAML, otherwise `<projectRoot>/.local-vector-search-mcp/index.db`. Relative knowledge base root and storage paths are resolved relative to the project root.

The server validates the configured root, non-empty include patterns, valid exclude patterns, storage path, search defaults, chunking limits, and embedding endpoint safety before starting indexing, CLI, or MCP operations. `chunking.maxChunkBytes` and `chunking.maxElements` must both be greater than zero. Include and exclude entries use filesystem glob semantics. The default include is `**/*.md`; the default exclude list is empty. CLI `--include` and `--exclude` are repeatable, and when either is present it replaces the corresponding YAML or default list instead of being merged. After path resolution, the root and storage path are absolute. Textual enum-like configuration values, including search default mode, are accepted case-insensitively and normalized to their canonical lowercase values when serialized.

All file access must stay inside the single configured project root. User-supplied paths are normalized, converted to absolute paths, checked for traversal, and verified to belong to that root using the platform's filesystem path case semantics. `kb_read` reads indexed content from SQLite and still validates path scope.

## Architecture And Patterns

Configuration records live in Core. YAML loading, CLI configuration parsing, merge, and path resolution live in Infrastructure. Invalid configuration raises `ConfigurationException`. Unauthorized path access raises `KnowledgeBaseAccessException`.

The embedding endpoint may be remote only when `allowRemoteEndpoint` is true. API keys, document text, embedding text, and raw vectors are never logged.

## Non-goals

JSON configuration, implicit global filesystem access, and path reads outside configured roots are excluded.

## Acceptance Criteria

- Explicit missing configuration files report `Configuration file was not found`.
- Missing `storage.path`, invalid `knowledgeBase.root`, empty include patterns, blank patterns, invalid search mode, invalid reindex scope, and unsafe remote embedding endpoints fail with clear messages.
- Include patterns select Markdown files using glob semantics, exclude patterns remove matching files using glob semantics, and invalid patterns fail with an understandable error.
- Without `--config`, the server starts from defaults, detects the project root, and does not require or implicitly read `local-vector-search-mcp.yml`.
- CLI `--root`, `--storage-path`, `--embedding-endpoint`, `--embedding-model`, `--include`, and `--exclude` override defaults and YAML values.
- Discovered document paths are project-relative, use `/` separators, and are returned in stable ordinal-ignore-case path order.
- A top-level `knowledgeBases` list is rejected with guidance to use singular `knowledgeBase`.
- Search default mode accepts `semantic`, `lexical`, or `hybrid` case-insensitively and serializes as lowercase.
- Chunking maximum bytes and maximum elements values of zero or less are rejected with clear configuration errors.
- `..` traversal and absolute paths are rejected.
- Loopback endpoints are accepted by default.
- Remote endpoints require `allowRemoteEndpoint: true`.

## Verification

- Unit-test configuration validation, enum-like value parsing, and path security.
- Start CLI commands with valid and invalid configuration files.
