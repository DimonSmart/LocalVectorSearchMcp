# 0002.spec-configuration-and-security

## Intent

The server is configured only through YAML in the MVP and must fail with understandable errors when configuration or path access is invalid.

## Related Specifications

- 0001.spec-product-overview.md
- 0004.spec-index-storage-and-search.md
- 0005.spec-mcp-and-cli.md

## Behavior

The default configuration file is `local-vector-search-mcp.yml`. A configuration path may be provided with `--config`, then through `LOCAL_VECTOR_SEARCH_MCP_CONFIG`, otherwise the current directory default is used.

Configuration includes server, storage, embedding, chunking, search, and a singular `knowledgeBase` section. `knowledgeBase.root` is required and names or lists of knowledge bases are not supported. A legacy top-level `knowledgeBases` section is rejected with a clear migration error. The default embedding endpoint is `http://localhost:11434/v1`, provider is `openai-compatible`, API key is required, model defaults to `bge-m3:latest`, batch size defaults to 16, and remote endpoints are rejected unless explicitly allowed.

The server validates the configured root, non-empty include patterns, valid exclude patterns, storage path, search defaults, chunking limits, and embedding endpoint safety before starting indexing, CLI, or MCP operations. After path resolution, the root is absolute. Textual enum-like configuration values, including search default mode, are accepted case-insensitively and normalized to their canonical lowercase values when serialized.

All file access must stay inside the single configured project root. User-supplied paths are normalized, converted to absolute paths, checked for traversal, and verified to belong to that root. `kb_read` reads indexed content from SQLite and still validates path scope.

## Architecture And Patterns

Configuration records live in Core. YAML loading and path resolution live in Infrastructure. Invalid configuration raises `ConfigurationException`. Unauthorized path access raises `KnowledgeBaseAccessException`.

The embedding endpoint may be remote only when `allowRemoteEndpoint` is true. API keys, document text, embedding text, and raw vectors are never logged.

## Non-goals

JSON configuration, implicit global filesystem access, and path reads outside configured roots are excluded.

## Acceptance Criteria

- Missing configuration files report `Configuration file was not found`.
- Missing `storage.path`, missing or invalid `knowledgeBase.root`, empty include patterns, blank patterns, invalid search mode, invalid reindex scope, and unsafe remote embedding endpoints fail with clear messages.
- A top-level `knowledgeBases` list is rejected with guidance to use singular `knowledgeBase`.
- Search default mode accepts `semantic`, `lexical`, or `hybrid` case-insensitively and serializes as lowercase.
- `..` traversal and absolute paths are rejected.
- Loopback endpoints are accepted by default.
- Remote endpoints require `allowRemoteEndpoint: true`.

## Verification

- Unit-test configuration validation, enum-like value parsing, and path security.
- Start CLI commands with valid and invalid configuration files.
