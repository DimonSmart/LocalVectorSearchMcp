# 0005.spec-mcp-and-cli

## Intent

The server exposes local knowledge base operations through official MCP stdio tools and provides CLI maintenance commands for reindexing and status checks.

## Behavior

Primary mode is MCP over stdio using the official `ModelContextProtocol` SDK. The server registers MCP tools from assembly and must not implement JSON-RPC, stdio transport, protocol messages, tool discovery, or schema generation manually.

The exposed tools are:

- `kb_reindex`: indexes or reindexes configured local Markdown knowledge bases.
- `kb_status`: returns local vector search index status.
- `kb_search`: searches the local Markdown knowledge base using lexical, semantic, or hybrid search.
- `kb_read`: reads indexed Markdown content from a document starting at a semantic pointer.

Tool classes are thin wrappers over application services. They must not contain SQL, embedding HTTP calls, Markdown parsing, chunking, or direct filesystem access.

CLI maintenance mode supports `--reindex` and `--status`. When either command is used, MCP stdio does not start and command output may go to stdout. In MCP mode logs go to stderr so stdout remains reserved for MCP transport.

## Architecture And Patterns

Server is a console application. It loads YAML configuration, registers application services, handles maintenance CLI commands, and otherwise starts the MCP stdio host with tools from assembly.

Tool inputs validate required query/path values, knowledge base names, reindex scopes, search modes, topK bounds, read limits, and pointer syntax. Search mode and reindex scope inputs accept their textual values case-insensitively and serialize outputs using canonical lowercase values. Tool outputs are simple serializable DTOs.

## Non-goals

Remote HTTP MCP, authentication, multi-user mode, background daemon mode, and custom MCP protocol implementation are excluded.

## Acceptance Criteria

- MCP server starts through official SDK stdio transport.
- Tools `kb_reindex`, `kb_status`, `kb_search`, and `kb_read` are exposed.
- Logs in MCP mode do not corrupt stdout.
- CLI `--reindex` and `--status` execute without starting MCP stdio.
- MCP tools return meaningful errors without normal stack traces.
- MCP search mode and reindex scope inputs accept valid values regardless of letter casing and serialize those values as lowercase.

## Verification

- Smoke-test DI and direct tool method calls.
- Unit-test MCP-facing enum-like input parsing and output serialization.
- Run CLI commands against a temporary knowledge base.
- Start the MCP server and confirm tool registration.
