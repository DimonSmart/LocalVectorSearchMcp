# 0001.spec-product-overview

## Intent

DimonSmart.LocalVectorSearchMcp is a local C#/.NET MCP server for indexing one project's configured Markdown root and searching it through lexical, vector, and hybrid retrieval.

The server targets `net10.0`, runs on Windows and Linux, uses `DimonSmart.LocalVectorSearchMcp` as the root namespace, and exposes knowledge base operations through MCP tools and maintenance CLI commands.

## Behavior

Each running server instance belongs to exactly one project, uses one configuration, indexes one allowed root into one project-local SQLite database, and cannot select or access another project's content through its API. Separate projects use separate server instances and indexes.

The product indexes only the configured root containing `.md` files. It parses Markdown into stable logical elements, builds semantic pointers, creates search chunks, stores index data in SQLite, and supports FTS5 lexical search, sqlite-vec vector search, and hybrid search through Reciprocal Rank Fusion.

The default search mode is hybrid. The server must provide MCP tools for reindexing, status, search, and reading indexed Markdown slices.

## Architecture And Patterns

The solution contains:

- `src/DimonSmart.LocalVectorSearchMcp.Core`
- `src/DimonSmart.LocalVectorSearchMcp.Infrastructure`
- `src/DimonSmart.LocalVectorSearchMcp.Server`
- `tests/DimonSmart.LocalVectorSearchMcp.Tests`
- `tests/DimonSmart.LocalVectorSearchMcp.IntegrationTests`

Core contains domain models, DTOs, validation, pure Markdown logic, semantic pointer logic, chunking, ranking, and service interfaces. Infrastructure contains SQLite, sqlite-vec, FTS5, YAML loading, file scanning, embedding HTTP calls, Markdown parser implementation, and indexing. Server is a console application that wires DI, CLI maintenance commands, and MCP stdio hosting.

## Non-goals

The MVP does not implement JSON config, PDF/DOCX import, web UI, file watcher, Git history indexing, reranking, a separate embedding cache table, remote HTTP MCP server, authentication, multi-user mode, background daemon mode, `kb_find_files`, a full knowledge base outline, advanced Markdown table/list/blockquote parsing, cloud storage, or embedding reuse across files by hash.

## Acceptance Criteria

- The solution builds with `dotnet build`.
- The solution builds and its non-manual tests pass on Linux.
- Non-manual tests pass with `dotnet test`.
- The server targets `net10.0`.
- The codebase contains no manual MCP protocol implementation.
- The product indexes Markdown files from the single configured project root into SQLite.
- A server instance exposes only its configured project; no API operation accepts a knowledge-base selector.
- Search works in lexical, semantic, and hybrid modes.
- Hybrid ranking uses Reciprocal Rank Fusion.
- Indexed content can be read by semantic pointer.

## Verification

- Run `dotnet build`.
- Run `dotnet test`.
- Run the Release build and non-manual tests in a Linux environment.
- Start the server in MCP stdio mode.
- Run CLI `--reindex` and `--status` against a test knowledge base.
